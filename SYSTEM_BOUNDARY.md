# System Boundary

## MegHQ
MegHQ is a distinct system.

It is:
- MeganCampaignHQ
- campaign operating system
- executive + staff operating structure
- doctrine / authority / execution / approval / desk system

## Not MegHQ
MegHQ is not:
- SkylerOS
- a generic orchestration backend
- a mixed campaign + separate OS repo
- a dumping ground for unrelated architecture

## Hard Separation Rule
SkylerOS and MegHQ are separate systems.
There should be no architectural overlap imported by assumption.
There should be no cross-contamination of canonical truth.

## Exception Rule
Only Megan-specific material accidentally written into the wrong place may be recovered into MegHQ.
Recovered material must be normalized into MegHQ canonical files.
The foreign system itself must not be imported.

## Future AI Rule
Do not propose merging the systems.
Do not "unify" them.
Do not infer shared architecture without explicit repo truth.
