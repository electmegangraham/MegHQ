# Input Capture System

## Purpose
All significant information entering the office should be captured, preserved, interpreted, and routed.

---

## Raw input capture
The office should support:
- typed input
- voice input
- uploads
- notes
- files
- structured forms
- event debriefs
- world awareness signals
- collaborator input

The original raw input should be preserved whenever useful.

---

## Signal extraction
From raw inputs, the system should extract:
- tasks
- ideas
- risks
- opportunities
- questions
- doctrine conflicts
- follow-ups
- relevant people or initiatives

---

## Routing logic
Inputs should be routed based on type and confidence.

Examples:
- operational instruction -> execution layer
- half-idea -> Brainstorm Studio
- uncertain but promising concept -> Development Lab
- risky high-impact what-if -> Simulation Engine
- campaign relevant fact -> department + memory + possibly doctrine

---

## Mental dump mode
A freeform mode for:
- “here’s everything in my head”
- post-event thoughts
- concerns
- random ideas
- memory capture

Mental dump mode should:
- preserve raw original
- extract structured outputs
- route items intelligently
- avoid losing nuance

---

## Automatic structuring
The system should be able to transform raw input into:
- tasks
- questions
- insights
- initiatives
- doctrine updates or conflicts
- desk-relevant items

This is one of the core bridges between conversation and operating system.
