# Megan's Desk System

## Purpose
Megan's Desk is the primary executive interface of Megan Campaign HQ.

It is not a dashboard-first experience.
It is a desk-first, conversation-first executive workspace where the Campaign Manager brings forward what matters, opens supporting material, and helps Megan direct the campaign without hunting through software.

---

## Why this matters
This is:
- the product experience
- the differentiator
- what Megan actually feels

If this is weak, the whole system feels like a dressed-up dashboard.
If this is strong, the system feels like a real virtual campaign office.

---

## Core principles

### One-screen-first model
The app should be designed around one primary screen:
- Megan's Desk

This desk is the default surface for:
- briefings
- approvals
- prep
- opportunities
- warnings
- schedule intelligence
- strong ideas
- visual context

### Conversation-first
The primary interaction is with the Campaign Manager via:
- voice
- typing
- clicking desk objects

### Hidden complexity
Dashboards, admin views, staff panels, analytics views, and structured operational interfaces should remain hidden by default.
They should surface only when:
- requested by Megan
- needed for context
- opened by the Campaign Manager to support a point

### Tell + show
The Campaign Manager should not only explain.
It should also show:
- charts
- schedules
- timelines
- memos
- comparisons
- initiative cards
- warnings
- prep materials
- visual artifacts

---

## Desk behaviors

### Open
Items can be opened onto the desk by:
- the Campaign Manager
- Megan request
- clicking an item already present

### Stack
Multiple items can coexist on the desk in a structured way.
Stacks should feel curated, not chaotic.

### Compare
The desk should support side-by-side comparison for:
- strategy options
- messaging options
- schedules
- donor opportunities
- risk tradeoffs

### Pin
Important items can be pinned so they remain visible across desk changes.

### Dismiss
Low-value or completed items can be dismissed without cluttering the workspace.

### File
Items can be filed into memory, topic areas, initiatives, departments, or later review.

### Revisit
Filed items can be surfaced later by:
- Campaign Manager recommendation
- Megan request
- related context
- schedule timing
- initiative reactivation

---

## What appears on the desk

Only items that are appropriate for executive attention should appear.

### Valid desk item categories
- decisions
- approvals
- awareness items
- prep packets
- opportunities
- warnings
- strategic prompts
- schedules with context
- briefing cards
- recommendation cards

### Invalid desk clutter
These should not appear by default:
- raw operational chatter
- weak drafts
- duplicate updates
- low-confidence noise
- generic dashboards
- unresolved low-value ambiguity

---

## Manager-driven UI
The Campaign Manager is the primary UI conductor.

It should be able to:
- open items
- close items
- expand items
- collapse items
- arrange items
- compare items
- highlight parts of an item
- bring up a supporting chart or memo
- surface a structured dashboard only when useful
- return the desk to a calm state afterward

---

## Check-In Mode
Check-In Mode must both tell and show.

### Tell
The manager explains:
- what is moving
- what is blocked
- what matters most
- what is emerging
- what needs Megan
- where bottlenecks may appear

### Show
The manager can open:
- a campaign status card
- a schedule view
- an initiative panel
- a chart
- a warning memo
- a donor opportunity
- a field summary
- a social/narrative snapshot

### Guide
The manager ends by clarifying:
- where Megan could help most
- where Megan should focus now
- what decisions need her
- what can wait

---

## Hidden dashboards principle
Structured dashboards still exist, but they are secondary.

They should be:
- hidden by default
- instantly accessible
- easy to dismiss
- non-dominant

They are for:
- power users
- staff mode
- diagnostics
- admin/config
- direct inspection when desired

---

## Interaction types

### Voice
Megan can:
- ask for updates
- request visuals
- give direction
- dump thoughts
- approve or reject
- ask follow-up questions

### Typing
Typing should support the same range as voice with more precision where needed.

### Click
Clicking should be lightweight and direct:
- open
- expand
- compare
- pin
- file
- approve
- reject
- delegate

---

## Desk composition rules
The Campaign Manager should compose the desk, not merely dump items onto it.

It should decide:
- what is open now
- what is collapsed
- what is side-by-side
- what is highlighted
- what is pinned
- what is deferred
- what is hidden

This is presentation intelligence, not just content surfacing.

---

## States the desk should support
- clean / idle
- briefing mode
- decision mode
- prep mode
- check-in mode
- review mode
- comparison mode
- urgent interruption mode

---

## Design intent
The desk should feel:
- calm
- premium
- intelligent
- anticipatory
- spatial
- conversational
- executive

It should not feel:
- like a cluttered CRM
- like a project management board
- like a noisy admin panel
- like a dashboard-first tool
