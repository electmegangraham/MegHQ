# User Baseline Profile System

## Purpose
The office should support uploading a user's baseline profile from day one so the system starts with a useful operating understanding before adaptation begins.

This is not limited to one user.
It should support baseline setup for Megan and, eventually, other real humans plugged into the office.

---

## What baseline profile can include
- personality traits
- communication preferences
- briefing preferences
- preferred workflow style
- tolerance for challenge vs support
- detail preference
- pacing preference
- executive style
- decision style
- priorities or aversions
- preferred output formats

---

## Why this matters
Without a baseline, the system starts too generic.
With a baseline, the office can immediately:
- brief better
- challenge better
- support better
- route better
- adapt faster

Then it can continue learning from real interaction over time.

---

## System behavior
The baseline should:
- initialize behavior
- influence briefing style
- influence assertiveness
- influence surfacing strategy
- influence workflow suggestions

Later, learning systems can refine beyond baseline.

---

## Principle
Day-one personalization + ongoing adaptation is stronger than waiting for the system to learn everything from scratch.
