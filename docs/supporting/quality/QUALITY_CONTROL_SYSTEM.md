# Quality Control System

## Purpose
The office cannot be allowed to merely answer prompts.
It must produce work that is:
- strategic
- deep
- useful
- coordinated
- campaign-ready
- doctrine-aligned

Quality must be a formal system, not a vibe.

---

## Core rule
No weak output reaches Megan.

---

## Quality gates

### Gate 1: Doctrine alignment
Does this match:
- campaign message
- tone
- strategic direction
- Megan preferences
- current goals

### Gate 2: Depth
Does this go deep enough?
- enough options
- enough consequences considered
- enough evidence
- enough thought

### Gate 3: Usefulness
Can this be used now?
- decision-ready
- action-ready
- campaign-ready
- briefing-ready

### Gate 4: Cross-department validation
Did it use the right upstream knowledge?
Examples:
- comms should use research
- fundraising should use relationships + schedule
- field should use doctrine + targeting + events
- social should use research + comms + strategy + world awareness

### Gate 5: Escalation / rewrite
If weak, shallow, generic, or under-evidenced:
- revise
- escalate
- route for stronger inputs
- do not quietly pass

---

## Gate outcomes
Each gate must support:
- pass
- raise issue
- ask / escalate ambiguity

This is required for:
- confusion
- missing inputs
- doctrine conflicts
- 50/50 choices
- low evidence
- dependency uncertainty

---

## Rewrite loops
Outputs that fail must not quietly disappear.
They should be:
- revised
- strengthened
- rerouted
- escalated

---

## Rejection criteria
Reject outputs that are:
- shallow
- generic
- filler-heavy
- fake polished
- strategically weak
- not actionable
- not grounded
- not aligned

---

## Confidence thresholds
Every major output should carry confidence awareness:
- low
- medium
- high

Low-confidence outputs should not be presented as certainty.

---

## Cross-department review
The office should support review layers:
- self-check
- peer/cross-review
- campaign manager review
- Megan review only when appropriate

Examples:
- research reviews factual claims
- strategy reviews strategic recommendations
- compliance/legal reviews risky items

---

## Output rubrics
Each output type should eventually have a rubric.

Example:
A briefing is not complete unless it includes:
- objective
- why it matters
- what Megan should do
- key risks
- key opportunities
- likely questions
- recommended line
- supporting materials

---

## Enforcement principle
Quality is not optional.
The system must be designed so low-grade output is structurally harder to pass than strong output.
