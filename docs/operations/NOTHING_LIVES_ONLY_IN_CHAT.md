# Nothing Lives Only In Chat

## Rule
If something meets any of these criteria, it belongs in the repo:

- affects behavior
- affects decision-making
- affects UI
- affects workflow
- affects quality
- affects collaboration

## Why
Under-saving causes drift.
System value should not remain trapped in conversation history.

## Application
If a concept materially changes how the campaign office behaves, it should be formalized into structured repo documentation.
