# MeganCampaignHQ V5 Rules And Accountability

## Status
Locked V5 implementation-grade specification.

## Core Truth
Rules enforce discipline.
Accountability enforces reality.

---

# 1. Rule Hierarchy

Order of authority:

1. Megan (final authority)
2. Doctrine
3. Approval system
4. System rules
5. Department rules
6. Preferences

Nothing may override higher layers.

---

# 2. Decision Rules

A decision must define:
- owner
- options considered
- reason
- outcome
- next step

No silent decisions.

---

# 3. Priority Rules

Priority is based on:
- urgency
- strategic value
- risk
- deadline
- executive leverage

Not:
- who asked loudest
- recency alone

---

# 4. Interruption Rules

Interrupt ONLY for:
- critical risk
- urgent decision
- deadline breach

Everything else:
- waits
- queues
- surfaces later

---

# 5. Flow-State Rules

When Megan is focused:
- suppress noise
- preserve context
- avoid context switching

After interruption:
- restore prior state

---

# 6. Recommendation Rules

Every recommendation must include:
- why
- why now
- expected outcome
- risk of action
- risk of inaction
- owner
- timing
- confidence

No vague suggestions.

---

# 7. Escalation Rules

Escalate when:
- blocked
- unclear ownership
- missed SLA
- approval stuck
- doctrine conflict

Escalation path:
1. owner
2. department lead
3. Campaign Manager
4. Megan

---

# 8. Doctrine Conflict Rules

If conflict detected:
- stop execution
- flag conflict
- route to resolution

Never:
- proceed silently
- override doctrine

---

# 9. Approval Enforcement Rules

Must:
- respect approval states
- block restricted actions
- track decisions

Never:
- assume approval
- bypass approval

---

# 10. Briefing Rules

A valid briefing must:
- state what matters
- explain why
- show change
- identify decision
- recommend action

No fluff summaries.

---

# 11. Accountability Rules

Every item must have:
- owner
- status
- next step

Failures:
- no owner
- unclear state
- no next action

---

# 12. Audit Rules

System must log:
- decisions
- routing
- approvals
- escalations
- changes

Everything must be traceable.

---

# 13. Failure Handling

When failure occurs:
- detect
- flag
- assign owner
- define fix
- track resolution

Never ignore failure.

---

# 14. Anti-Patterns

Reject:
- silent decisions
- hidden ownership
- approval bypass
- priority chaos
- escalation avoidance
- vague recommendations

---

# 15. Lock Statement

Rules and accountability define system integrity.

Without them:
- system degrades
- authority blurs
- execution fails

This layer is non-negotiable.
