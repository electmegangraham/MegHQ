# MeganCampaignHQ V5 Taste Website Governance

## Status
Locked implementation-grade specification.

## Core Truth
Taste is not optional.
Website quality is strategic.
If it feels like a generic campaign, it is wrong.

---

# 1. Purpose

This subsystem defines:
- taste rules
- anti-pattern rules
- website governance
- homepage hierarchy
- CTA logic
- trust-engine behavior
- approved vs rejected tone

---

# 2. Final Taste Rule

Preferred feeling:
- calm
- official
- credible
- structured
- trust-first
- polished
- grounded

Rejected feeling:
- playful
- bubbly
- loud
- cheesy
- generic campaign-y
- dashboard-like
- cluttered
- overdesigned

---

# 3. We Must Preserve

- above-the-fold clarity
- strong hero presence
- trust-first layout
- simple clear hierarchy
- grounded language
- professional systems tone
- campaign as well-run office
- clean spacing
- controlled typography
- premium but restrained visuals

---

# 4. We Must Reject

- oversized playful text
- bubbly UI
- cartoonish energy
- cluttered hero sections
- too many competing calls to action
- generic political slogans
- flashy dashboard surfaces
- fake CRM feel
- feature sprawl
- noisy overlays
- fluff language

---

# 5. Website Core Rule

The website is not a brochure.
It is a trust engine.

It must:
- establish seriousness
- establish credibility
- establish readiness
- convert attention into action

---

# 6. Homepage Hierarchy

Homepage priority order:

1. trust / credibility
2. Megan identity
3. why she is running
4. office / systems competence
5. action path
6. supporting content

No disorder.

---

# 7. Above-The-Fold Rule

Above the fold must communicate:
- who Megan is
- what office
- seriousness
- trust
- one clear action path

No clutter.
No overstacking.
No confusion.

---

# 8. Hero Rule

Hero must feel:
- presidential
- official
- composed
- strong
- not gimmicky

Hero must not feel:
- stock-like
- playful
- amateur
- overlaid with too much noise

---

# 9. CTA Governance

CTA hierarchy:
- one primary CTA
- optional secondary CTA
- no CTA overload

Primary CTA must be clear.
Secondary CTA must support, not compete.

---

# 10. Header Governance

Header must be:
- stable
- clean
- trust-building
- readable
- persistent

Header must not become:
- cluttered
- oversized
- visually loud

---

# 11. Section Governance

Each section must have:
- one purpose
- one hierarchy level
- clear spacing
- strong readability

No filler sections.
No decorative complexity.

---

# 12. Copy Governance

Approved copy characteristics:
- grounded
- direct
- serious
- competent
- human
- clear

Rejected copy characteristics:
- fluffy
- vague
- over-marketed
- slogan-heavy
- generic political

---

# 13. Page Existence Rule

A page should exist only if it:
- builds trust
- answers a real question
- moves conversion
- supports credibility
- supports field execution

No unnecessary pages.

---

# 14. Feature Sprawl Rule

Do not add features just because they are available.

Reject:
- unnecessary dashboards
- unnecessary maps
- decorative widgets
- dead-end forms
- placeholder surfaces
- complexity with no trust gain

---

# 15. Visual Governance

Visual system must favor:
- clean contrast
- disciplined spacing
- restrained color use
- professional imagery
- strong legibility

No visual gimmicks.

---

# 16. Desk / Product Taste Crossover

The same taste rules apply across the platform:
- calm authority
- high signal
- restrained design
- anti-clutter
- premium not flashy

Website and product must feel like the same brain.

---

# 17. Failure Modes

- site feels generic
- hero loses authority
- CTA hierarchy breaks
- copy becomes fluffy
- pages multiply without purpose
- visuals become noisy
- trust weakens

Must be blocked.

---

# 18. Lock Statement

Taste and website governance are now enforceable system truth.

No future build may:
- drift into generic campaign design
- trade trust for noise
- add feature sprawl
- weaken calm authority
