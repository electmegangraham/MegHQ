# V4 Continuity State

## Purpose
This file captures the current continuity state of Megan Campaign HQ V4 so the system can be resumed without drift.

---

## What V4 is
V4 is a virtual campaign office.

It is not:
- a dashboard-first tool
- a simple CRM
- a generic admin console

It is:
- desk-first
- conversation-first
- campaign-manager-driven
- department-backed
- multimodal
- premium
- designed to feel like a real campaign office

---

## Locked truths
- Megan is final authority
- Campaign Manager drives execution
- Doctrine / Holy Page is governing truth
- Megan's Desk is the primary interface
- Dashboards are secondary and hidden by default
- Departments are real operating lanes
- Departments can be AI, human, or hybrid via seats
- Quality is a formal system
- The office should learn and adapt
- Nothing important should live only in chat

## Product experience rule
Default experience should be:
- one desk
- campaign manager conversation
- items surfaced when needed
- visuals shown in context
- deeper interfaces only when requested or useful

---

## Anti-drift rule
If a future change contradicts:
- desk-first
- manager-driven
- Megan final authority
- doctrine enforcement
- no weak output reaches Megan
- hidden complexity by default

then it should be treated as drift unless explicitly approved as a new direction.
